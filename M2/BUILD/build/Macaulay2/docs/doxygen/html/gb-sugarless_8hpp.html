Detailed Description

GBinhom_comp — Buchberger GB without the sugar heuristic, primarily for inhomogeneous input.

Declares GBinhom_comp, a GBComputation subclass that runs the same Buchberger loop as gbA but selects S-pairs purely by leading-monomial degree instead of by sugar (the degree the S-polynomial would have if no cancellation occurred). The class works on homogeneous modules too but is principally intended for inhomogeneous inputs where sugar-aware selection could mislead, and as a sugar-free baseline for regression testing the default gbA. The header comment is candid that performance is "poor in many cases, dramatically good in others" — a worthwhile fallback rather than a general default.

It plugs into the same GBComputation interface as the rest of the GB strategy family and is reachable through M2's Strategy => argument. The homogeneous specialisation (gb-homog2.hpp), the toric variant (gb-toric.hpp), and the Groebner walk (gb-walk.hpp) cover the remaining non-default paths.
