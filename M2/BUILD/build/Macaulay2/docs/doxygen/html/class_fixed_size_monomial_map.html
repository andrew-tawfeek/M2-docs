Detailed Description

template<class T>
class FixedSizeMonomialMap< T >

Concurrent hashtable mapping from monomials to T with a fixed number of buckets. Lookups are lockless while insertions grab a lock.

There is no limitation on the number of entries that can be inserted, but performance will suffer if the ratio of elements to buckets gets high.

You can insert new values but you cannot change the value that an already-inserted value maps to. It is possible to clear the table but this operation is not safe for concurrency.
