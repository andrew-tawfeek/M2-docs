Detailed Description

template<class T>
class Atomic< T >

This class is equivalent to std::atomic<T>. Some functions from the interface of std::atomic are missing - add them as necessary. Do not add operator= and operator T() — it is better to make the code explicit about when and how loading and storing of atomic variables occurs.

The purpose of the class is that it performs far better than std::atomic for some implementations. For example the std::atomic in MSVC 2012 performs a compare-and-swap operation on a load even with the paramter std::memory_order_relaxed.

We force all the functions to be inline because they can contain switches on the value of std::memory_order. This will usually be a compile-time constant parameter so that after inlining the switch will disappear. Yet the code size of the switch may make some compilers avoid the inline.
