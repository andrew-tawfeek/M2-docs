Detailed Description

Variable-length sparse (variable, exponent) encoding of monomials.

Each monomial is stored as [len, v_1, e_1, ..., v_r, e_r] with len = 2r + 1 and the variable indices in strictly decreasing order (v_1 > v_2 > ... > v_r >= 0, all e_i > 0). The layout is dense in support but sparse in the ambient variable count, so monomials of low support stay compact and divisibility / equality reduce to a single linear walk through the canonical form. An ExponentList is a view — it does not own its storage; the caller may back it with a std::vector<int>, a contiguous Range, or a MemoryBlock bump-pointer arena depending on lifetime requirements.

Conceptually a parallel encoding lives in f4/varpower-monomial.hpp for the F4 inner loop, and the dense counterpart is ExponentVector.hpp. Newer gb-f4/ code is consolidating these onto its own MonomialView / MonomialTypes; this header remains the workhorse at the top of e/ (and the encoding shared with monideal.hpp).
