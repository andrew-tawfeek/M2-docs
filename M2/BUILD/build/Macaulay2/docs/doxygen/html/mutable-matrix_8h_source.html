311M2_bool rawLLL(MutableMatrix *M,
312 MutableMatrix /* or null */ *U,
313 gmp_QQ threshold,
314 int strategy);
315/* DAN: connected to rawLLL */
316/* Given a mutable matrix M over ZZ, and a rational number threshold, 1/4 <
317 threshold <= 1, modify M so that the columns form a Lenstra-Lenstra-Lovasz
318 basis of the image of (the original) M. ASSUMPTION: (strategy=0 case)
319 the columns of M are already a a basis for the lattice.
320 The algorithm used is that in Cohen's book on computational algebraic
321 number theory, BUT: beware of the typos in the algorithm! If there is any
322 error (interrupted, M or threshold not the correct kind), then false is
323 returned, and LLL is set to 0. If M has a column change of basis matrix
324 attached, it will be modified accordingly.
325
326 strategy: 0 means use original Macaulay2 engine routine.
327 2 means use NTL LLL
328 (strategy%3) == 3 means use one of the real number variants:
329 GramSchmidt or Givens: 0 or 4 (respectively)
330 LLL or BKZ: 0 or 8 (respectively)
331 FP, QP1, QP, XD, RR (respectively: 0, 16, 2*16, 3*16, 4*16
332 Thus: strategy 3+4+8+16 means use NTL's GBKZ_QP1.
333
334 For the RR variants, the suggested value of the threshold is 99/100.
335
336 If U is not NULL, then it should be an m by m matrix over ZZ, where m is the
337 number of columns of the matrix M. In this case, it is set to be the
338 invertible transform matrix such that Mold * U = Mnew.
339*/
