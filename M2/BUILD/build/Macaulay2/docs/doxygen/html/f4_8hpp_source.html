6// My implementation of Faugere's linear algebra GB routines. Also includes
7// free resolution code.
8
9// Template parameters include:
10// coefficient ring arithmetic
11// packed_monomial
12// exponents
13// varpower_monomial
14// Types to define
15// MonomialLookupTable
16// a. find_divisor(packed_monomial, comp) --> integer whose lead term
17// divides packed_monomial
18// b. insert(packed_monomial, comp, index)
19// packed_monomial
20// implemented as packed exponent vector, perhaps with weight vector(s).
21// monomial compare is done on a order by order basis (i.e. a different
22// function)
23// each packed_monomial also includes its hash code.
24// multiplication: checked, and unchecked.
25// quotient, lcm, gcd?
26// coefficient arithmetic
27// coeff types include: ZZ/p, GF(q), ZZ, QQ, QQ[x]/f(x), other tower
28// extensions?
29// kk(a,...,z)
30// this is done at "low" level: we try to do as many ZZ ops as possible
31// before reducing mod p.
32// required ops: add, mult, negate, is_zero, divide(in ZZ/p,GF), what else?
33// also required: translation to/from M2 ring_elem.
34// spair
35// spairs and gens
36// make_spair, make_gen
37// where to store the packed_monomial's?
38// SPairSet
39// insert spair
40// prepare for next degree
41// next spair
42// remove redundant pairs
43// Minimalize a set of spairs
44// Used in finding new spairs.
45// Creation of the matrix
46// This is the heart of the matter.
47// Need:
48// hashtable for column monomials (packed_monomial,comp)'s.
49// sort the list of columns
50// column header
51// row header
52// several types of rows:
53// (a) one that is essentially a list of monomials
54// (b) one after arithmetic has been performed
55// how should these rows be implemented?
56// row reduction of the matrix (using ZZ/p to get QQ).
57// rows --> new gb elements
58// gb itself
59// generators
60//
61// syzygies via this method
62// minimalization of these syzygies
63//
64// keep in mind:
65// skew commutative multiplication
66// Schreyer orders
67// quotient rings
68// Hilbert function use
69//
